================================================
HANDBOOK MQTT - INTEGRASI JAM PINTAR & BACKEND
================================================

1. KONEKSI BROKER MQTT
Pihak IoT (Smartwatch) diwajibkan untuk terhubung ke MQTT Broker milik Backend menggunakan protokol WebSockets Secure (WSS).
- Protokol : mqtts / wss
- Host/URL : wss://iot-broker.xianly.cloud
- Port     : 443
- Client ID: Boleh diisi dengan MAC Address Jam.
- Auth     : (Kosong / Anonymous untuk saat ini).

2. ALUR PENGUKURAN (TOPIC & PAYLOAD)
Terdapat 2 jenis Topic yang digunakan. Ganti <MAC_ADDRESS> dengan nomor seri / MAC Address dari masing-masing jam.

A. Topic Streaming Data PPG (Detak Jantung)
Saat sensor menyala dan membaca detak jantung, kumpulkan datanya (buffering) ke dalam Array lalu kirim secara streaming setiap sekian milidetik.
- Topic  : watch/<MAC_ADDRESS>/stream
- Method : PUBLISH (QoS 0)
- Format : Array JSON (Pastikan tidak ngirim angka satu-satu!)
- Contoh Payload (Body):
  [240, 245, 230, 210, 250]

B. Topic Status / Command (Start & Stop)
Topic ini digunakan untuk komunikasi dua arah antara Jam dan Backend terkait status pengukuran.
- Topic  : watch/<MAC_ADDRESS>/status
- Method : PUBLISH (QoS 1) & SUBSCRIBE

Skenario Trigger:
1. Jam harus melakukan SUBSCRIBE ke topic `watch/<MAC_ADDRESS>/status` sejak awal dinyalakan.
2. Jika Backend mengirimkan payload berupa text "START", firmware Jam harus merespon dengan cara menyalakan sensor dan mulai nge-publish data ke topic Stream (Poin A).
3. Jika pengukuran sudah selesai atau Backend mengirimkan "STOP", firmware Jam harus mematikan sensor.
4. Jam lalu diwajibkan melakukan PUBLISH text "FINISHED" ke topic status ini sebagai pertanda ke backend bahwa file rekaman sudah boleh diproses/di-save oleh Database AI.

================================================
Catatan Tambahan untuk IoT Developer:
Tolong pastikan format saat nge-publish Stream itu benar-benar Array `[]`. Jika payload yang diterima Backend bukan Array JSON, sistem Backend akan nge-drop/mengabaikan paket data tersebut.
